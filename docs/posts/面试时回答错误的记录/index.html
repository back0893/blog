面试时回答错误的记录

2026-07-23
Go 面试

不应该的 这些都是go的基础 我居然忘记了，面试时错误或者乱回答，真是不应该

panic

func Test()(result string){
    defer func(){
        result = "defer"
    }()  
    panic("panic")
    return "fail"
}

  • 应该是返回什么？ 初始回答是defer，缺少了对panic的处理导致回答错误

  • 正确回答和原因 没有返回 触发panic，如果defer里有recory去捕获，那么返回"defer"

channle

对于关闭的channle

  • 对其发送数据 会触发panic
  • 对其接收数据 会返回零值和false(我忘记了 记住 _,isOpen:=<-channel) 第二个值 false时表示channel已关闭

slice

a0 := []int{1,2,3}
a1 := a0[1:3]
a1[0] = 100
log.Println(a0)

输出 [1,100,3]

我的错误点

  1. 忘记了 切片是左闭右开 我认为a1[1:3]会导致panic 我忘记了slice元素的范围取值
  2. 不是 [100,2,3] 我也不知道 我为什么会这样回答 明明就是a1取值的底层数组不会变 当a1修改了底层数组的元素 会导致a0的元素也会被修改

并发

counter:=0
wg:=sync.WaitGroup{}
for i:=0;i<1000;i++{
    wg.Add(1)
    go func(i int){
        defer wg.Done()
        counter++
    }(i)
}
wg.Wait()
log.Println(counter)

输入： 小于1000的整数 我知道并发会导致counter的值不正确 但是我为什么没有回答正确？

  1. 认为一定要回答一个字 而不是不确定 我应该回答不确定
  2. 因为协程并发 会导致counter++ 可能出现同一个值被重复加一 比如 2个协程都还是10+1

总结

  • 这些基础知识 我居然忘记了，面试时错误或者乱回答，真是不应该
  • 对于使用ai生成的代码变多后 这些基础的一些常识或者语法规则 我应该多去看看书或者文档
